# Обработка ошибок и авто-перезапуск

## Что было добавлено

### 1. Глобальный фильтр исключений
**Файл:** `apps/api/src/common/filters/all-exceptions.filter.ts`

Фильтр перехватывает все необработанные исключения и:
- Для API запросов (`/v1/*`, `/api/*`) возвращает JSON с деталями ошибки
- Для браузерных запросов возвращает красивую HTML страницу с:
  - Кодом ошибки
  - Детальными логами (timestamp, method, path, error)
  - Кнопками "На главную" и "Перезагрузить"
  - Автоматической перезагрузкой через 10 секунд

### 2. Обработка процессов в main.ts
**Файл:** `apps/api/src/main.ts`

Добавлены обработчики:
- `unhandledRejection` - логирует незавершенные обещания
- `uncaughtException` - логирует неперехваченные исключения

### 3. Менеджер авто-перезапуска
**Файл:** `restart-server.js`

Скрипт запускает API и автоматически перезапускает его при падении:
- Максимум 5 быстрых перезапусков (если падает сразу)
- Перезапуск через 3 секунды после падения
- Логирование всех событий в `logs/restart-manager.log`
- Корректная обработка SIGINT/SIGTERM

## Использование

### Запуск с авто-перезапуском
```bash
npm run start:api:watch
```

### Запуск без авто-перезапуска
```bash
npm run start:api
```

### В режиме разработки
```bash
npm run dev:api
```

## Ярлык на рабочем столе

Ярлык `Nodir HDD Fixer` на рабочем столе теперь запускает сервер с авто-перезапуском.

## Логи

Все логи сохраняются в папку `logs/`:
- `app.log` - основные логи приложения
- `error.log` - только ошибки
- `restart-manager.log` - логи менеджера перезапуска

## Пример HTML страницы ошибки

При ошибке 404 на браузерный запрос:
```
http://localhost:3004/
```

Вернется красивая страница с:
- Красным заголовком "ERROR 404"
- Детальной информацией об ошибке
- Логами с timestamp, method, path
- Кнопками действий
- Авто-перезагрузкой через 10 секунд

## Пример JSON ответа для API

```json
{

  "path": "/v1/unknown",
  "method": "GET",
  "message": "Not Found",
  "error": "Not Found"

```
