# 🤖 Настройка Telegram уведомлений

## 📋 Шаг 1: Создание Telegram бота

### 1.1 Откройте BotFather

Перейдите в Telegram и найдите **@BotFather** или откройте:
```
https://t.me/BotFather
```

### 1.2 Создайте нового бота

Отправьте команду:
```
/newbot
```

### 1.3 Введите имя бота

BotFather попросит ввести имя:
```
RecoveryUZ Bot
```

### 1.4 Введите username бота

Username должен заканчиваться на `bot`:
```
RecoveryUZBot
```

### 1.5 Скопируйте токен

BotFather выдаст токен вида:
```
1234567890:ABCdefGHIjklMNOpqrsTUVwxyz
```

**⚠️ Сохраните токен в надёжном месте!**

---

## 📍 Шаг 2: Узнайте свой Chat ID

### Вариант 1: Через бота

1. Откройте **@userinfobot**
2. Нажмите Start
3. Бот отправит ваш Chat ID

### Вариант 2: Вручную

1. Отправьте любое сообщение вашему новому боту
2. Откройте в браузере:
```
https://api.telegram.org/bot<ВАШ_ТОКЕН>/getUpdates
```
3. Найдите в ответе `"chat":{"id":123456789}`

### 3.1 Создайте файл .env.production

```bash
cd /home/mrnurali/nodir_hdd_fixer
cp .env.production.example .env.production
```

## 🆘 Troubleshooting

### Бот не отправляет сообщения

**Проверьте переменные:**
```bash
cat .env.production | grep TELEGRAM
```

**Проверьте токен:**
```bash
curl https://api.telegram.org/bot<ВАШ_ТОКЕН>/getMe
```

**Проверьте Chat ID:**
```bash
curl https://api.telegram.org/bot<ВАШ_ТОКЕН>/getUpdates
```

### Сообщение не приходит

1. Убедитесь что вы отправили боту `/start`
2. Проверьте что бот не заблокирован
3. Проверьте логи API:
```bash
docker compose -f docker-compose.prod.yml logs api
```

### Ошибка "bot was blocked"

Пользователь заблокировал бота. Разблокируйте и отправьте `/start`.

## 📝 Дополнительные настройки

### Добавить бота в канал

1. Создайте канал в Telegram
2. Добавьте бота как участника
3. Назначьте бота администратором
4. Узнайте Chat ID канала (через @GetMyIDBot)
5. Обновите `TELEGRAM_CHAT_ID`

### Настроить уведомления для разных событий

Создайте разные каналы:
- `@RecoveryUZDeploy` - деплой
- `@RecoveryUZOrders` - заказы
- `@RecoveryUZErrors` - ошибки
