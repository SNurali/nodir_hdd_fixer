# 🧠 AI Context: HDD Fixer (nodir-hdd-fixer)

Этот файл создан для сохранения контекста ИИ-ассистента по проекту HDD Fixer.

## 📌 О Проекте
Система управления сервисным центром по ремонту оборудования (жесткие диски, SSD, серверы и др.).
Архитектура проекта — монорепозиторий на базе **Turborepo**.

## 🏗 Конфигурация и Структура
- `apps/api/` — Backend: **NestJS 10**, TypeScript, **TypeORM**, **PostgreSQL 16**, BullMQ, кастомное логгирование (Pino). Авторизация через JWT + OAuth2.
- `apps/web/` — Frontend: **Next.js**, React, TailwindCSS.
- `packages/shared/` — Общие ресурсы: типы, эннамы и валидация (`Zod`).

## 🛠 Инструкция по запуску (Commands)
```bash
# Инициализация
docker compose up -d
npm install

# База данных
npm run db:migrate
npm run db:seed

# Разработка и проверки
npm run dev
npm run build
npm run test
npm run lint
```
*Инфрасруктура API доступна на порту 3001, Frontend — 3000.*

## ⚠️ Правила работы с кодом
1. **Безопасность и Валидация:** Все новые DTO создавать в `packages/shared` с помощью `Zod`. Строжайшим образом защищать API-эндпоинты лимитами (`Throttler`) и `JwtAuthGuard`, кроме эндпоинтов с декоратором `@Public()`.
2. **База Данных:** Никогда не менять БД в обход миграций `apps/api/src/database/migrations`.
3. **Логгирование:** Использовать инжектированный `PinoLogger` для важных бизнес-операций.
4. **Язык общения:** Отвечать юзеру на его языке (преимущественно русский), кратко и по делу без лишней воды. Проверять код (`lint`/`test`/`build`) перед коммитом или сдачей задачи.
