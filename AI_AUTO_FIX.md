# 🔧 AI Auto-Fix System

## Что это?

Система **автоматического запуска Qwen AI** при падении приложения. Если сервер падает 3 раза подряд, автоматически запускается Qwen для анализа и исправления ошибки.

## Как работает?

```
┌─────────────────────────────────────────────────────────┐
│  1. Запуск приложения                                   │
│     ↓                                                   │
│  2. Падение? → Перезапуск                               │
│     ↓                                                   │
│  3. Снова падение? → Перезапуск                         │
│     ↓                                                   │
│  4. Еще падение? → 🤖 ЗАПУСК QWEN AI!                   │
│     ↓                                                   │
│  5. Qwen получает логи и отчет об ошибке               │
│     ↓                                                   │
│  6. Qwen анализирует и исправляет код                   │
│     ↓                                                   │
│  7. Qwen пересобирает проект                            │
│     ↓                                                   │
│  8. Автоматический перезапуск                           │
└─────────────────────────────────────────────────────────┘
```

## Использование

### Запуск с AI-помощником
```bash
npm run start:api:ai
```

### Запуск с обычным авто-перезапуском
```bash
npm run start:api:watch
```

### Обычный запуск
```bash
npm run start:api
```

## Что делает Qwen при падении?

1. **Получает автоматический запрос** с:
   - Кодом ошибки (exit code, signal)
   - Последними 50 строками из `app.log`
   - Последними 50 строками из `error.log`
   - Временем падения
   - Путем к отчету о падении

2. **Анализирует логи** и находит причину

3. **Исправляет код** в файлах проекта

4. **Пересобирает** проект (`npm run build:api`)

5. **Сообщает** что было исправлено

## Пример отчета о падении

Файл: `logs/crash-report-1234567890.json`

```json
{
  "timestamp": "2026-03-05T19:30:00.000Z",
  "exitCode": 1,
  "signal": null,
  "crashes": [...],
  "appLog": "...",
  "errorLog": "..."
}
```

## Логи

- `logs/restart-manager.log` - логи менеджера перезапуска
- `logs/app.log` - логи приложения
- `logs/error.log` - только ошибки
- `logs/crash-report-*.json` - отчеты о падениях

## Настройки

В файле `restart-server-with-ai.js`:

```javascript
const MAX_RESTARTS_BEFORE_AI = 3;  // Через сколько падений звать AI
const RESTART_DELAY = 3000;        // Задержка перед перезапуском (мс)
const AI_HELP_DELAY = 5000;        // Задержка после помощи AI (мс)
```

## Сценарии использования

```
Падение → Перезапуск → Падение → Перезапуск →

### ⚠️ Сценарий 3: Qwen не смог исправить

Падение → ... → Qwen AI → Не смог исправить →

```

## Отключение AI-помощника

Если хотите использовать обычный рестарт без AI:

1. Откройте ярлык на рабочем столе
2. Измените строку:
   ```
   Exec=gnome-terminal ... --command="npm run start:api:watch"
   ```

Или используйте в терминале:
```bash
npm run start:api:watch
```
