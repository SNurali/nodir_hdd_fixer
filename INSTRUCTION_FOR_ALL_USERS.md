# 📖 ИНСТРУКЦИЯ ПО РАБОТЕ С ЗАКАЗАМИ
## Полное руководство для всех пользователей

---

## 🎯 КРАТКАЯ СХЕМА (5 шагов)

```
1. Заказ создан
   ↓
2. Диагностика (мастер)
   ↓
3. Цена выставлена → ЖДЁМ ОДОБРЕНИЯ КЛИЕНТОМ ⭐
   ↓
4. Ремонт (только после одобрения!)
   ↓
5. Выдача заказа
```

**⚠️ ГЛАВНОЕ ПРАВИЛО:** Без одобрения цены клиентом ремонт начинать **НЕЛЬЗЯ**!

---

## 👥 ДЛЯ КЛИЕНТОВ

### 📝 Как создать заказ?

1. Откройте сайт
2. Нажмите **"Новый заказ"**
3. Выберите:
   - Устройство (например, "Жёсткий диск")
   - Проблему (например, "Не включается")
4. Оставьте контакты (имя, телефон)
5. Нажмите **"Отправить"**

✅ **Готово!** Вы получите уведомление.

---

### 🔔 Что происходит дальше?

| Шаг | Статус | Кто действует | Что делать вам |
|-----|--------|---------------|----------------|
| 1 | 🟣 В ожидании | Админ принимает | Ждать |
| 2 | 🔵 Принят | Мастер диагностирует | Ждать |
| 3 | 🔷 Диагностика | Мастер выставляет цену | Ждать уведомления |
| 4 | 🟠 **Ожидает согласования** | **ВЫ!** ⭐ | **Одобрить цену!** |
| 5 | 🟢 Одобрено | Мастер начинает ремонт | Ждать |
| 6 | 🟡 В работе | Мастер ремонтирует | Ждать |
| 7 | 🟩 Завершён | Готов к выдаче | Оплатить и получить |

---

### ⭐ ОДОБРЕНИЕ ЦЕНЫ — САМОЕ ВАЖНОЕ!

Когда вы получите уведомление **"Цена выставлена"**:

#### Вариант 1: Цена устраивает
1. Откройте заказ
2. Нажмите **"✅ Одобрить цену"**
3. Мастер начнёт ремонт

#### Вариант 2: Цена НЕ устраивает
1. Откройте заказ
2. Нажмите **"❌ Отклонить цену"**
3. Укажите причину (например, "Дорого")
4. Мастер пересчитает цену

**⚠️ Помните:** Без вашего одобрения ремонт **НЕ начнётся**!

---

### 📱 Как отслеживать заказ?

1. Зайдите в личный кабинет
2. Откройте **"Мои заказы"**
3. Нажмите на заказ
4. Смотрите:
   - Текущий статус
   - Прогресс по шагам
   - Историю изменений

## 👨‍🔧 ДЛЯ МАСТЕРОВ

### 📋 Пошаговая инструкция

#### Шаг 1: Принятие заказа
**Кто:** Админ или Оператор  
**Действие:**
```
PATCH /orders/:id
{ "status": "accepted" }
```

---

#### Шаг 2: Начало диагностики
**Кто:** Вы (Мастер)  
**Действие:**
```
PATCH /orders/:id
{ "status": "diagnosing" }
```

**Что делать:**
- ✅ Проведите полную диагностику
- ✅ Определите все неисправности
- ✅ Запишите обнаруженные проблемы

---

#### Шаг 3: Установка цены ⭐
**Кто:** Вы (Мастер), Админ, Оператор  
**Действие:**
```
POST /orders/:id/set-price
{
  "details": [
    { "detail_id": "...", "price": 500000 }
  ]
}
```

**⚠️ ВАЖНО:**
- Статус **автоматически** изменится на `🟠 Ожидает согласования`
- Клиент получит уведомление
- **Теперь ждём одобрения клиента!**

---

#### Шаг 4: Ожидание одобрения ⏳

**Статус:** `🟠 Ожидает согласования`

**Что делать:**
- ⏳ Ждать пока клиент одобрит цену
- 📧 Можно напомнить клиенту
- ❌ **НЕЛЬЗЯ начинать ремонт до одобрения!**

---

#### Шаг 5: Начало ремонта
**Кто:** Вы (Мастер)  
**Условие:** Статус `🟢 Одобрено`  
**Действие:**
```
PATCH /orders/:id
{ "status": "in_progress" }
```

**⚠️ ПРОВЕРЬТЕ:**
- ✅ Цена одобрена клиентом (статус `approved`)
- ✅ Все запчасти есть в наличии
- ✅ Можно начинать работу

---

#### Шаг 6: Завершение ремонта
**Кто:** Вы (Мастер)  
**Действие:**
```
PATCH /orders/:id
{ "status": "completed" }
```

**Что сделать перед завершением:**
- ✅ Выполнить все работы
- ✅ Установить запчасти
- ✅ Протестировать устройство

---

#### Шаг 7: Выдача заказа
**Кто:** Админ или Оператор  
**Действие:**
```
PATCH /orders/:id
{ "status": "issued" }
```

---

## 👔 ДЛЯ АДМИНИСТРАТОРОВ

### 📊 Управление заказами

#### 1. Принятие заказа
```
PATCH /orders/:id
{ "status": "accepted" }
```

#### 2. Назначение мастера
```
POST /orders/:id/assign-master
{ "master_id": "uuid-мастера" }
```

#### 3. Контроль этапов

| Этап | Статус | Контроль |
|------|--------|----------|
| Диагностика | `diagnosing` | Мастер проводит диагностику |
| Цена | `pending_approval` | Ждём одобрения клиента |
| Ремонт | `in_progress` | Мастер выполняет работы |
| Завершение | `completed` | Проверка качества |
| Выдача | `issued` | Финальное закрытие |

#### 4. Работа с проблемными заказами

**Клиент долго не одобряет цену:**
- 📧 Отправить напоминание
- 📞 Позвонить клиенту
- ⚠️ Предложить альтернативы

**Мастер долго не выставляет цену:**
- ⚠️ Напомнить мастеру
- 📋 Проверить загрузку мастера
- 🔄 Переназначить мастера

## 🚨 КРИТИЧЕСКИ ВАЖНЫЕ ПРАВИЛА

### ⭐ Правило №1: ОДОБРЕНИЕ ЦЕНЫ

```
❌ НЕПРАВИЛЬНО:
Диагностика → В работе
           ↑ 
      (без одобрения!)

✅ ПРАВИЛЬНО:
Диагностика → Ожидает согласования → Одобрено → В работе
                            ↑
                    (клиент одобряет)
```

---

### ⭐ Правило №2: Последовательность

**НЕЛЬЗЯ перепрыгивать этапы!**

```
❌ new → in_progress (нельзя!)
✅ new → accepted → diagnosing → pending_approval → approved → in_progress
```

---

### ⭐ Правило №3: Уведомления

**Автоматические уведомления:**

| Событие | Кто получает | Текст |
|---------|-------------|-------|
| Цена выставлена | Клиент | "Цена за ваш заказ: X UZS. Одобрите." |
| Цена одобрена | Админ, Мастер | "Клиент одобрил цену. Можно работать." |
| Статус изменён | Клиент | "Статус заказа: {статус}" |

## 📞 ЧАСТЫЕ ВОПРОСЫ

### ❓ Клиент: "Сколько ждать диагностику?"
**Ответ:** Обычно 1-2 часа после принятия заказа

### ❓ Клиент: "Можно начать ремонт без одобрения?"
**Ответ:** ❌ Нет, это нарушает процесс

### ❓ Мастер: "Клиент не одобряет цену, что делать?"
**Ответ:** 
1. Позвонить клиенту
2. Отправить напоминание
3. Предложить альтернативы

### ❓ Админ: "Можно ли изменить цену после одобрения?"
**Ответ:** 
- ✅ Можно, но нужно новое одобрение клиента
- ⚠️ Старое одобрение аннулируется
